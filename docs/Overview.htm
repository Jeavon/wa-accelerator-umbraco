Windows Azure Accelerator for Umbraco

The Windows Azure Accelerator for Umbraco is designed to enable Umbraco applications to be easily deployed into the Windows Azure. By running in Windows Azure, Umbraco applications can benefit from the automated service management, reduced administration, high availability, and scalability provided by the Windows Azure Platform. For more information about the benefits of Windows Azure, please visit the Windows Azure web site.


The accelerator has been designed to enable you to rapidly deploy Umbraco applications and updates to your application without redeploying a full Windows Azure Service Package.


How does it work?

The accelerator is an application which synchronizes the instance of Umbraco in the Hosted Service with those stored in Blob Storage. This synchronization occurs both ways, so that any change done on an instance is replicated to all the others.

If a new instance of Umbraco is detected in blob storage, then a new site is created and configured in the Hosted Service IIS, leveraging the Full IIS feature introduced with Windows Azure SDK & Tools 1.3.

Prerequisites

For running the setup scripts and generating the deployment package you will need the following pre-requisites:

  • Visual Studio 2010 SP1 and .NET 4
  • Windows Azure SDK and Tools for Visual Studio (November 2011) version 1.6
  • SQL Express 2008 R2

You will also need the following Windows Azure resources to deploy Umbraco sites:

  • Hosted service running at least 1 extra small instance (2 recommended minimum)
  • 1 SQL Azure Web edition database, 1GB (or larger if needed)
  • Azure storage account

If you decide to store ASP.NET session state in SQL Azure (currently the accelerator wizard default), then you will need to use an additional Web edition 1GB database.

If you decide to store session state using Azure Caching, you will need to setup a 128MB cache.
